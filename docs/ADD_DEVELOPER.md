# 👥 Добавление Разработчика в Проект - Инструкция для Администратора

## 🚀 Быстрое добавление разработчика

### Шаг 1: Получение информации от разработчика

Попросите разработчика предоставить:
- **GitHub username** (например: `@username`)
- **Email** для связи
- **Желаемую роль** (Developer, Maintainer, Admin)

### Шаг 2: Добавление в GitHub репозиторий

#### Вариант A: Через веб-интерфейс GitHub

1. Откройте репозиторий: `https://github.com/your-org/nutry_flow`
2. Перейдите в **Settings** → **Collaborators and teams**
3. Нажмите **Add people** или **Invite a collaborator**
4. Введите GitHub username разработчика
5. Выберите уровень доступа:
   - **Read** - только чтение (для ревьюеров)
   - **Write** - чтение и запись (для разработчиков)
   - **Maintain** - управление репозиторием (для tech leads)
   - **Admin** - полный доступ (для владельцев)
6. Отправьте приглашение

#### Вариант B: Через GitHub CLI

```bash
# Установите GitHub CLI (если еще не установлен)
brew install gh  # macOS
# или
sudo apt install gh  # Linux

# Авторизуйтесь
gh auth login

# Добавьте разработчика
gh api repos/:owner/:repo/collaborators/:username \
  -X PUT \
  -f permission=write
```

### Шаг 3: Отправка приветственного сообщения

Отправьте разработчику следующее сообщение:

```
Привет! 👋

Добро пожаловать в проект NutryFlow!

Ваш доступ к репозиторию настроен. Следующие шаги:

1. Клонируйте репозиторий:
   git clone https://github.com/your-org/nutry_flow.git
   cd nutry_flow

2. Запустите скрипт настройки:
   ./scripts/setup_developer_environment.sh

3. Прочитайте документацию:
   - QUICK_START_DEVELOPER.md - быстрый старт (5 минут)
   - docs/developer-onboarding-guide.md - полное руководство

4. Заполните .env файл (скопируйте из .env.example)

Если возникнут вопросы, обращайтесь в Slack: #nutryflow-dev

Удачи! 🚀
```

## 🔐 Уровни доступа

### Developer (Write)
**Рекомендуется для большинства разработчиков**

- ✅ Чтение кода
- ✅ Создание веток
- ✅ Создание Pull Requests
- ✅ Комментирование Issues
- ❌ Слияние в main ветку (только через PR)
- ❌ Удаление веток

### Maintainer
**Для Tech Leads и Senior разработчиков**

- ✅ Все права Developer
- ✅ Слияние Pull Requests
- ✅ Управление Issues
- ✅ Управление Labels
- ✅ Удаление веток
- ❌ Удаление репозитория

### Admin
**Только для владельцев проекта**

- ✅ Все права Maintainer
- ✅ Управление настройками репозитория
- ✅ Управление правами доступа
- ✅ Удаление репозитория

## 🛡️ Безопасность

### Важные правила:

1. **Никогда не давайте Admin доступ** без необходимости
2. **Проверяйте GitHub профиль** перед добавлением
3. **Используйте двухфакторную аутентификацию** (2FA) для всех
4. **Регулярно проверяйте список** разработчиков
5. **Удаляйте неактивных** разработчиков

### Защита веток:

- **main** - защищена, требует Pull Request и review
- **develop** - защищена, требует Pull Request
- **feature/*** - свободный доступ для разработчиков
