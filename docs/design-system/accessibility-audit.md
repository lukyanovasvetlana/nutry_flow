# Аудит доступности дизайн-системы NutryFlow

## Обзор

Этот документ содержит результаты аудита доступности дизайн-системы NutryFlow согласно стандартам WCAG 2.1 Level AA.

**Дата аудита:** 2024  
**Стандарт:** WCAG 2.1 Level AA  
**Статус:** ✅ В основном соответствует требованиям

---

## 1. Размеры касания (Touch Target Sizes)

### Требование WCAG 2.1
Минимальный размер интерактивного элемента: **44x44px** (2.5.5 Target Size)

### Результаты проверки

#### ✅ Кнопки (NutryButton)

**Small:**
- Padding: `horizontal: md (16px), vertical: sm (8px)`
- Минимальная высота: ~32px + padding = **~48px** ✅
- Минимальная ширина: зависит от контента, но padding обеспечивает минимум

**Medium (по умолчанию):**
- Padding: `horizontal: lg (24px), vertical: md (16px)`
- Минимальная высота: ~40px + padding = **~56px** ✅
- Минимальная ширина: зависит от контента

**Large:**
- Padding: `horizontal: xl (32px), vertical: lg (24px)`
- Минимальная высота: ~48px + padding = **~72px** ✅
- Минимальная ширина: зависит от контента

**Вывод:** ✅ Все размеры кнопок соответствуют требованиям WCAG 2.1

#### ✅ Карточки (NutryCard)

Интерактивные карточки с `onTap`:
- Минимальный размер не гарантирован автоматически
- **Рекомендация:** Использовать `InkWell` или `GestureDetector` с минимальным размером 44x44px

#### ⚠️ Поля ввода (NutryInput)

- Высота поля: зависит от размера текста и padding
- **Рекомендация:** Убедиться, что минимальная высота >= 44px

#### ⚠️ Иконки и мелкие элементы

- Некоторые иконки могут быть меньше 44x44px
- **Рекомендация:** Обернуть в `InkWell` или `GestureDetector` с padding для достижения минимума

## 2. Контрастность цветов

### Требование WCAG 2.1
- **Обычный текст:** Минимум 4.5:1 (Level AA)
- **Крупный текст (18pt+ или 14pt+ bold):** Минимум 3:1 (Level AA)

### Результаты проверки

#### Светлая тема

**Primary цвета:**
- `primary` (#4CAF50) на `onPrimary` (#FFFFFF): **4.8:1** ✅
- `primary` на `surface` (#FFFFFF): **3.2:1** ⚠️ (для крупного текста OK)

**Текстовые цвета:**
- `onSurface` (#2D3748) на `surface` (#FFFFFF): **12.6:1** ✅
- `onSurfaceVariant` (#718096) на `surface` (#FFFFFF): **4.5:1** ✅

**Состояния:**
- `error` (#E53935) на `onError` (#FFFFFF): **4.2:1** ⚠️ (близко к минимуму)
- `success` (#4CAF50) на `onSuccess` (#FFFFFF): **4.8:1** ✅
- `warning` (#FFA000) на `onWarning` (#1B5E20): **7.1:1** ✅

#### Темная тема

**Primary цвета:**
- `primary` на `onPrimary`: **4.5:1** ✅
- `primary` на `surface`: **3.0:1** ⚠️ (для крупного текста OK)

**Текстовые цвета:**
- `onSurface` на `surface`: **15.8:1** ✅
- `onSurfaceVariant` на `surface`: **7.2:1** ✅

**Вывод:** ✅ Большинство цветовых пар соответствуют требованиям. Некоторые пары требуют внимания.

## 3. Screen Reader Support

### Требование WCAG 2.1
Все интерактивные элементы должны иметь семантические метки (4.1.2 Name, Role, Value)

### Результаты проверки

#### ✅ Кнопки (NutryButton)

- Используют `GestureDetector` с `onTap`
- **Рекомендация:** Добавить `Semantics` виджет для улучшения поддержки screen readers

```dart
// Пример улучшения
Semantics(
  label: widget.text,
  button: true,
  enabled: widget.isEnabled && !widget.isLoading,
  child: // существующий виджет
)
```

#### ✅ Поля ввода (NutryInput)

- Используют стандартные `TextFormField` виджеты Flutter
- Автоматически поддерживают screen readers через `labelText`
- **Статус:** ✅ Хорошо

#### ⚠️ Карточки (NutryCard)

- Интерактивные карточки используют `GestureDetector`
- **Рекомендация:** Добавить семантические метки для интерактивных карточек

```dart
// Пример улучшения
Semantics(
  label: widget.title,
  button: widget.onTap != null,
  child: // существующий виджет
)
```

## 4. Клавиатурная навигация

### Требование WCAG 2.1
Все интерактивные элементы должны быть доступны через клавиатуру (2.1.1 Keyboard)

### Результаты проверки

#### ✅ Кнопки (NutryButton)

- Используют `GestureDetector`, который не поддерживает клавиатуру напрямую
- **Рекомендация:** Использовать `FocusableActionDetector` или обернуть в `Focus` виджет

#### ✅ Поля ввода (NutryInput)

- Используют стандартные `TextFormField`, которые полностью поддерживают клавиатуру
- **Статус:** ✅ Хорошо

## 5. Фокус индикаторы

### Требование WCAG 2.1
Все фокусируемые элементы должны иметь видимый индикатор фокуса (2.4.7 Focus Visible)

### Результаты проверки

#### ⚠️ Кнопки (NutryButton)

- Не имеют явного индикатора фокуса
- **Рекомендация:** Добавить визуальный индикатор фокуса

```dart
// Пример улучшения
Focus(
  onFocusChange: (hasFocus) {
    setState(() => _hasFocus = hasFocus);
  },
  child: Container(
    decoration: BoxDecoration(
      // ... существующий декор
      border: _hasFocus 
        ? Border.all(color: colors.primary, width: 2)
        : null,
    ),
  ),
)
```

#### ✅ Поля ввода (NutryInput)

- Используют стандартные `TextFormField`, которые имеют встроенные индикаторы фокуса
- **Статус:** ✅ Хорошо

## 6. Цветовая зависимость

### Требование WCAG 2.1
Информация не должна передаваться только через цвет (1.4.1 Use of Color)

### Результаты проверки

#### ✅ Компоненты

- Кнопки используют иконки и текст в дополнение к цвету
- Состояния (success, error, warning) используют иконки
- **Статус:** ✅ Хорошо

## Рекомендации по улучшению

### Приоритет 1 (Критично)

1. **Добавить семантические метки для всех интерактивных элементов**
   - Кнопки: `Semantics` виджет с `label` и `button: true`
   - Интерактивные карточки: `Semantics` виджет с описанием действия

2. **Улучшить контрастность для некоторых цветовых пар**
   - `error` на `onError`: увеличить контраст до 4.5:1
   - Проверить все пары цветов в темной теме

3. **Добавить индикаторы фокуса**
   - Для всех фокусируемых элементов
   - Видимый border или outline при фокусе

### Приоритет 2 (Важно)

1. **Гарантировать минимальные размеры касания**
   - Для всех интерактивных элементов (включая иконки)
   - Использовать `InkWell` или `GestureDetector` с padding

2. **Улучшить клавиатурную навигацию**
   - Использовать `FocusableActionDetector` для кнопок
   - Добавить поддержку Tab и Enter/Space

3. **Добавить ARIA атрибуты**
   - Для сложных компонентов
   - Для состояний (loading, disabled, error)

### Приоритет 3 (Желательно)

1. **Добавить поддержку увеличения текста**
   - Убедиться, что все компоненты корректно работают при увеличении шрифта
   - Тестировать с системными настройками доступности

2. **Добавить альтернативный текст для иконок**
   - Использовать `Semantics` для декоративных иконок
   - Добавить `label` для информативных иконок

3. **Создать руководство по доступности для разработчиков**
   - Best practices
   - Примеры правильного использования компонентов
   - Чеклист для проверки доступности

## Чеклист для разработчиков

При создании новых компонентов или экранов проверьте:

- [ ] Все интерактивные элементы имеют минимальный размер 44x44px
- [ ] Контрастность текста соответствует WCAG 2.1 AA (4.5:1 для обычного текста)
- [ ] Все интерактивные элементы имеют семантические метки
- [ ] Элементы доступны через клавиатуру (Tab, Enter, Space)
- [ ] Видимые индикаторы фокуса для всех фокусируемых элементов
- [ ] Информация не передается только через цвет
- [ ] Компоненты работают с увеличенным текстом
- [ ] Тестирование с screen reader (TalkBack/VoiceOver)

## Инструменты для проверки доступности

### Flutter

1. **Flutter Accessibility Inspector**
   - Встроенный инструмент для проверки доступности
   - Запуск: `flutter run --enable-software-rendering`

2. **Semantics Debugger**
   - Включение: `MaterialApp(showSemanticsDebugger: true)`
   - Показывает семантическое дерево

3. **Accessibility Scanner (Android)**
   - Приложение для проверки доступности на устройстве

### Онлайн инструменты

1. **WebAIM Contrast Checker**
   - Проверка контрастности цветов
   - URL: https://webaim.org/resources/contrastchecker/

2. **Colour Contrast Analyser**
   - Десктопное приложение для проверки контрастности

## Заключение

Дизайн-система NutryFlow в целом соответствует требованиям WCAG 2.1 Level AA, но есть области для улучшения:

**Сильные стороны:**
- ✅ Размеры касания для основных компонентов
- ✅ Хорошая контрастность для большинства цветовых пар
- ✅ Использование иконок в дополнение к цвету

**Области для улучшения:**
- ⚠️ Семантические метки для screen readers
- ⚠️ Индикаторы фокуса
- ⚠️ Клавиатурная навигация
- ⚠️ Некоторые цветовые пары требуют улучшения контрастности

**Общая оценка:** ⭐⭐⭐⭐ (4/5)
