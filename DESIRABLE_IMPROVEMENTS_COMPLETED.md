### 3. ✅ Улучшена обработка ошибок парсинга

**Проблема:** Отсутствие явной обработки ошибок при парсинге числовых значений.

**Решение:**
- Добавлены явные проверки на `null` после `tryParse`
- Добавлены понятные сообщения об ошибках
- Улучшена валидация для всех числовых полей

**Улучшенная валидация:**
```dart
// Было:
final height = int.tryParse(value);
if (height == null || height < _minHeight || height > _maxHeight) {
  return 'Рост должен быть от $_minHeight до $_maxHeight см';
}

// Стало:
final height = int.tryParse(value);
if (height == null) {
  return 'Введите корректное значение роста';
}
if (height < _minHeight || height > _maxHeight) {
  return 'Рост должен быть от $_minHeight до $_maxHeight см';
}
```

**Улучшено для полей:**
- ✅ Рост (int)
- ✅ Вес (double)
- ✅ Целевой вес (double)
- ✅ Калории (int)
- ✅ Белки (double)
- ✅ Углеводы (double)
- ✅ Жиры (double)

**Результат:**
- ✅ Более понятные сообщения об ошибках
- ✅ Разделение ошибок парсинга и валидации диапазона
- ✅ Улучшенный UX при вводе данных
