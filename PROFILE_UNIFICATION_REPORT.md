# Отчет об объединении экранов профиля

**Дата:** 2025-01-27  
**Задача:** Объединить дублирующиеся экраны профиля  
**Статус:** ✅ Завершено

---

## 📋 Резюме

Успешно объединены два дублирующихся экрана профиля (`profile_screen.dart` и `profile_view_screen.dart`) в единый экран `UnifiedProfileScreen` с использованием TabBar для переключения между просмотром профиля и настройками.

---

## 🎯 Проблема

### До объединения

**Два отдельных экрана:**
1. **`profile_screen.dart`** - экран настроек
   - Простой header с аватаром
   - Редактирование профиля
   - Переключение темы
   - Настройки уведомлений
   - Смена пароля
   - Выход из аккаунта
   - Политика конфиденциальности
   - ❌ Нет BLoC, нет состояний загрузки/ошибок
   - ❌ Простая загрузка данных из Supabase

2. **`profile_view_screen.dart`** - экран просмотра профиля
   - Использует BLoC паттерн
   - ProfileHeader с градиентом
   - ProfileInfoCard, ProfileStatsCard, ProfileGoalsCard
   - Быстрые действия
   - ✅ Хорошие состояния загрузки/ошибок
   - ✅ Pull-to-refresh
   - ❌ Требует userId как параметр

**Проблемы:**
- Дублирование функциональности
- Пользователь может запутаться, какой экран использовать
- Нет четкого разделения ответственности
- Разные подходы к управлению состоянием
- Несогласованность в использовании дизайн-токенов

---

## ✅ Решение

### Создан объединенный экран `UnifiedProfileScreen`

**Ключевые особенности:**

1. **TabBar навигация**
   - Вкладка "Профиль" - просмотр информации о профиле
   - Вкладка "Настройки" - конфигурация профиля и приложения

2. **Единое управление состоянием**
   - Использует BLoC паттерн для управления состоянием профиля
   - Автоматическое получение userId из текущего пользователя Supabase
   - Полная поддержка состояний: Loading, Error, Loaded, Updating, Updated

3. **Унифицированный дизайн**
   - Использует дизайн-токены везде (`DesignTokens`, `ThemeTokens`)
   - Консистентные отступы, радиусы, цвета
   - Единый стиль карточек через `NutryCard`

4. **Объединенная функциональность**

   **Вкладка "Профиль":**
   - ProfileHeader с градиентом и базовой информацией
   - ProfileInfoCard - личная информация
   - ProfileStatsCard - статистика здоровья с BMI
   - ProfileGoalsCard - цели и прогресс
   - Быстрые действия (редактировать, фото)
   - Pull-to-refresh для обновления данных

   **Вкладка "Настройки":**
   - Редактирование профиля
   - Смена пароля
   - Переключение темы (светлая/темная)
   - Настройки уведомлений (Push, Email, Еженедельные отчёты)
   - Выход из аккаунта
   - Версия приложения
   - Политика конфиденциальности
   - Условия использования

5. **Улучшенный UX**
   - Состояния загрузки с индикаторами
   - Обработка ошибок с возможностью повтора
   - Pull-to-refresh для обновления данных
   - Модальные окна для выбора фото
   - Диалоги подтверждения для критических действий

## 🎨 Архитектурные улучшения

### 1. Единое управление состоянием

**До:**
```dart
// profile_screen.dart - нет состояний
Future<void> _loadUserData() async {
  // Простая загрузка без состояний
}

// profile_view_screen.dart - BLoC
_profileBloc.add(LoadProfile(userId));
```

**После:**
```dart
// UnifiedProfileScreen - единый BLoC
BlocConsumer<ProfileBloc, ProfileState>(
  builder: (context, state) {
    if (state is ProfileLoading) return LoadingWidget();
    if (state is ProfileError) return ErrorWidget();
    if (state is ProfileLoaded) return ProfileContent();
  },
)
```

### 2. Унифицированные дизайн-токены

**До:**
```dart
// Смешение стилей
color: Colors.grey[50]  // ❌
color: AppColors.primary  // ✅
borderRadius: BorderRadius.circular(12)  // ❌
```

**После:**
```dart
// Единые токены везде
backgroundColor: context.background  // ✅
color: context.primary  // ✅
borderRadius: BorderRadius.circular(DesignTokens.borders.md)  // ✅
padding: EdgeInsets.all(DesignTokens.spacing.md)  // ✅
```

### 3. Автоматическое получение userId

**До:**
```dart
// profile_view_screen.dart - требует userId как параметр
ProfileViewScreen({required this.userId})
```

**После:**
```dart
// UnifiedProfileScreen - автоматически получает userId
final user = SupabaseService.instance.currentUser;
_userId = user?.id;
```
