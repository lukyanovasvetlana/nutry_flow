## 🔴 Критические проблемы

### 1. **Неправильная интеграция FormField в ProfileSelectionField**
**Проблема:** `ProfileSelectionField` использует `FormField` с `initialValue`, но значение не обновляется при изменении `value` извне.

**Расположение:** `profile_selection_field.dart:36-37`

**Проблемный код:**
```dart
FormField<T>(
  initialValue: value,  // ❌ Это значение устанавливается только один раз
  validator: validator,
```

**Решение:** Использовать `FormField` с контроллером или обновлять через `didUpdateWidget`.

---

### 2. **ProfileMultiSelectionField не интегрирован с Form**
**Проблема:** `ProfileMultiSelectionField` не использует `FormField`, поэтому его валидация не работает с общей формой.

**Расположение:** `profile_edit_screen.dart:423-434, 436-457, 459-478, 576-599`

**Последствия:**
- Невозможно валидировать множественные выборы
- Ошибки не отображаются при валидации формы
- Нет интеграции с `_formKey.currentState!.validate()`

---

### 3. **Неправильный парсинг роста**
**Проблема:** Рост парсится как `double`, но должен быть `int` (сантиметры - целое число).

**Расположение:** `profile_edit_screen.dart:773-775`

**Проблемный код:**
```dart
height: _heightController.text.isEmpty
    ? null
    : double.tryParse(_heightController.text),  // ❌ Должно быть int.tryParse
```

**Решение:** Использовать `int.tryParse` и привести к `double` при необходимости.

---

### 4. **Неработающая прокрутка к ошибке**
**Проблема:** `_scrollToFirstError()` всегда прокручивает к началу (0), а не к первому полю с ошибкой.

**Расположение:** `profile_edit_screen.dart:804-811`

**Проблемный код:**
```dart
void _scrollToFirstError() {
  _scrollController.animateTo(
    0,  // ❌ Всегда прокручивает к началу
    duration: const Duration(milliseconds: 300),
    curve: Curves.easeInOut,
  );
}
```

**Решение:** Найти первое поле с ошибкой через `GlobalKey` и прокрутить к нему.

---

## ⚠️ Важные проблемы

### 5. **Дублирование валидации**
**Проблема:** Валидация дублируется в UI и в UseCase (`update_user_profile_usecase.dart`).

**Последствия:**
- Дублирование кода
- Риск расхождения правил валидации
- Сложность поддержки

**Рекомендация:** Вынести валидацию в отдельный сервис или использовать единый источник истины.

---

### 6. **Отсутствие debouncing для отслеживания изменений**
**Проблема:** Каждое изменение в текстовом поле вызывает `setState` через `_onFormChanged()`.

**Расположение:** `profile_edit_screen.dart:74-84, 111-117`

**Последствия:**
- Избыточные перерисовки
- Потенциальные проблемы с производительностью при быстром вводе

**Решение:** Добавить debouncing для `_onFormChanged()`.

---

### 7. **Жестко заданные цвета в сообщениях**
**Проблема:** Используются `Colors.green` и `Colors.red` вместо цветов темы.

**Расположение:** `profile_edit_screen.dart:850-866`

**Проблемный код:**
```dart
ScaffoldMessenger.of(context).showSnackBar(
  const SnackBar(
    content: Text('Профиль успешно обновлен'),
    backgroundColor: Colors.green,  // ❌ Жестко заданный цвет
  ),
);
```

**Решение:** Использовать `AppColors.dynamicSuccess` и `AppColors.dynamicError`.

---

### 8. **Проблема с валидацией в ProfileSelectionField**
**Проблема:** `FormField` в `ProfileSelectionField` не обновляет значение при изменении через `onTap`, поэтому валидация может работать некорректно.

**Расположение:** `profile_selection_field.dart:36-93`

**Детали:** `FormField` использует `initialValue`, но когда значение изменяется через диалог, `FormField` не знает об этом изменении.

---

### 9. **Отсутствие обработки ошибок парсинга**
**Проблема:** При парсинге числовых значений нет явной обработки ошибок.

**Расположение:** `profile_edit_screen.dart:764-799`

**Пример:**
```dart
height: _heightController.text.isEmpty
    ? null
    : double.tryParse(_heightController.text),  // Может вернуть null, но нет явной проверки
```

**Рекомендация:** Добавить явные проверки и логирование.

---

### 10. **Дублирование описания в диалоге уровня активности**
**Проблема:** В диалоге выбора уровня активности `displayName` отображается дважды.

**Расположение:** `profile_edit_screen.dart:722-738`

**Проблемный код:**
```dart
Text(
  level.displayName,  // Первый раз
  style: TextStyle(...),
),
Text(
  level.displayName,  // ❌ Второй раз (дублирование)
  style: TextStyle(...),
),
```

**Решение:** Второй `Text` должен показывать описание, а не `displayName`.

## 📋 Проблемы качества кода

### 11. **Слишком много контроллеров в одном виджете**
**Проблема:** 11 `TextEditingController` в одном State-классе.

**Последствия:**
- Сложность поддержки
- Риск утечек памяти
- Сложность тестирования

**Рекомендация:** Вынести управление формой в отдельный класс или использовать пакет `reactive_forms`.

---

### 12. **Магические числа**
**Проблема:** Жестко заданные значения валидации разбросаны по коду.

**Примеры:**
- `age < 13 || age > 120` (строки 312, 102)
- `height < 100 || height > 250` (строки 353, 109)
- `weight < 30 || weight > 300` (строки 374, 116)
- `calories < 800 || calories > 5000` (строка 496)

**Решение:** Вынести в константы или конфигурационный класс.

---

### 13. **Отсутствие семантических меток**
**Проблема:** Нет `Semantics` виджетов для accessibility.

**Последствия:**
- Плохая поддержка screen readers
- Низкая доступность приложения

---

### 14. **Отсутствие управления фокусом**
**Проблема:** Нет автоматического перехода фокуса между полями.

**Рекомендация:** Использовать `FocusNode` для навигации по форме.

---

### 15. **Потенциальная утечка памяти в ProfileMultiSelectionField**
**Проблема:** `TextEditingController` в диалоге не освобождается явно.

**Расположение:** `profile_multi_selection_field.dart:124`

**Проблемный код:**
```dart
final TextEditingController customController = TextEditingController();
// ❌ Нет dispose()
```

**Решение:** Добавить `dispose()` в `StatefulBuilder` или использовать `dispose` callback.
